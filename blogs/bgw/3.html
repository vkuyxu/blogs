    # 4.null 和 undefined 的区别？

    undefined 代表的是含义是未定义，null 代表的含义是空对象。

    变量声明未定义会返回 undefined，null 主要用于赋值给一些可能会返回对象的变量，作为初始化。

    undefined 不是保留字，可以用来当做变量名，这会影响对 undefined 值的判断。可以使用 void 0 来获得安全的 undefined

    # 6.为什么 0.1+0.2 不等于 0.3，如何让其相等？

    0.1 和 0.2 转换为二进制都是无限循环的数，所以 0.1+0.2 其实是等于 0.30000000000000004。

    解决办法:设置一个误差范围，这个值通常为 2 的负 52 次方，在 Es6 中提供了 Number.EPSILON 属性，它

    的值就是 2 的-52 次方，所以我们只需要判断 0.1+0.2-0.3 是否小于 Number.EPSILON，如果小于，就返

    回 true

    # 8.isNaN 和 Number.isNaN 函数的区别？

    # 9.||和&&操作符的返回值。

    对于||来说，如果条件判断为 true，就返回第一个操作数的值，如果为 false 就返回第二个操作数的值。

    对于&&来说，如果条件判断为 true，就返回第二个操作数的值，如果为 false 就返回第一个操作数的值。

    ||和&&返回它们其中一个操作数的值，而不是条件判断的结果

    # 12.map 和 object 的区别?

    map:

    1. 默认情况下不包含任何键，只包含显示插入的键。

    2. map 的键可以是任意值，包括函数，对象或者任意基本类型。

    3. map 的 key 是有序的。

    4. map 的键值对个数可以通过 size 属性获取

    5. map 可以直接被迭代。

    6. 在频繁增删键值对的情况下性能更好。

    object：

    1. object 有一个原型，原型链上面的键名可能和自己设置的键名冲突。

    2. object 的键名必须是 string 或者 Symbol

    3. object 的键是无序的

    4. object 的键值对个数需要手动计算

    5. object 需要获取键值之后才能迭代。

    6. 频繁增删键值对的情况下没有优化。

    # 13.map 和 weakMap 的区别?

    map：

    本质上是键值对的集合，但是键名不限制类型，可以是任意类型。

    1. get 方法，读取 key 对应的键值，如果找不到 key，返回 undefined。

    2. set 方法，设置键名对应的键值，返回 map 结构，如果 key 已经有值，则更新，否则就会生成一个新的键值对。

    3. has 方法，判断是否包含某个键名，返回布尔值。

    4. delete 方法，删除某个键，成功返回 true，失败返回 false

    5. clear 方法，清空所有成员

    weakmap:

    键名必须是对象，没有clear方法。weakMap 键名所指的对象不计入垃圾回收机制。

    # 15.javascript 脚本延迟加载的方式有哪些？

    2. async 属性，给 js 添加 async 属性，使 js 脚本异步加载，加载完毕后立即执行。但是如果在 js 脚本加载完毕时，页面还没有加载完成,这时会阻塞页面的加载。

    3. 动态创建 dom 的方式来加载脚本，监听文档的加载事件，在文档加载完毕后，动态创建 srcipt 标签来引入 js 脚本。

    4. 将 js 脚本放在文档的最底部。

    # 18.什么是 BOM 和 DOM？

    DOM 指的是文档对象模型，它指的是将文档当做一个对象，这个对象主要定义了处理网页内容的方法和接口。

    BOM 指的是浏览器对象模型。它指的是将浏览器作为一个对象，这个对象主要定义了与浏览器进行交互的方法和接口。

    BOM 的核心是 window，window 不仅是一个用来访问浏览器窗口的一个接口，又是一个全局的对象。

    这意味着在网页中定义的任何对象，变量和函数，都将作为全局对象的一个属性或者方法存在。

    # 20.JavaScript 为什么要进行变量提升，它导致了什么问题?

    变量提升就是将无论在什么地方声明的变量都提升到作用域的顶部，可以在声明前访问。

    优点：解析和预编译过程中的声明提升可以提高性能，让函数可以在执行时预先为变量分配栈空间

    声明提升还可以提高 JS 代码的容错性，使一些不规范的代码也可以正常执行

    # 21.ES6 模块与 CommonJS 模块有什么异同?

    1. 语法差异

    ES6 模块使用 import 关键字导入模块，使用 export 关键字导出模块或模块中的特定内容。

    CommonJS 模块使用 require() 函数导入模块，使用 module.export 或 export 对象导出模块或模块中的

    特定内容

    1. 加载时机差异

    ES6 模块在编译时静态解析，可以在代码执行之前进行模块加载。

    CommonJS 模块在运行时动态加载，模块加载发生在代码执行阶段。

    1. 导出方式的差异

    ES6 模块可以导出多个值，并且支持导出默认值和命名导出。

    CommonJS 模块只能导出一个默认值。

    1. 导入方式的差异

    ES6 模块的导入是静态的，CommonJS 模块的导入是动态的

    # 22.use strict 是什么意思 ? 使用它区别是什么?

    开启严格模式。

    目的：提高效率，增加编译速度。减少 js 的不合理之处，不严谨之处，减少怪异行为。保证代码的运行安全

    # 24.ajax、axios、fetch 的区别

    ajax实现了网页的局部更新。但是是基于 mvc 实现的，不符合 mvvm 的浪潮，且有回调地狱问题。

    Fetch是基于 promise 设计的，不是 ajax 的进一步封装，而是原生 js，没有使用 XMLHttpRequest 对象。

    优点:

    • 语法简洁，更加语义化。

    • 支持 async/await

    • api 更加丰富

    缺点:

    • 只对网络请求报错，对 400,500 都当做成功的请求。

    • 默认不携带 coolie。

    Axios是基于 promise 封装的 http 客户端。

    • 可以在多平台运行，支持 promiseAPI

    • 监听请求和返回，

    • 自动转换 json 数据。

    • 客户端抵御 XSRF 攻击。

    # 25.对于原型和原型链的理解

    在 JavaScript 中，每个对象都有一个特殊的属性称为原型（prototype）。原型是一个对象，它包含共享属性和方法的定义。当我们访问对象上的属性或方法时，如果该对象本身没有该属性或方法，JavaScript 引擎会尝试在原型链中查找。

    原型链是一种链式结构，由对象的原型组成。每个对象都有一个隐式指向其原型的链接，通过 proto 属性（ES6 之前）或 Object.getPrototypeOf(obj) 方法（ES6+）可以访问。当我们访问一个对象的属性时，JavaScript 首先在对象本身查找该属性。如果没有找到，则会继续在对象的原型上查找。如果原型也没有该属性，JavaScript 会继续在原型的原型上查找，以此类推，直到找到该属性或到达原型链的末尾（即对象的原型为 null）。

    # 27.对闭包的理解

    闭包指的是有权限访问另一个函数作用域中变量的函数，创建闭包最常见的方式就是在函数内部创建另一个函数，创建的函数可以访问到当前函数的局部变量

    常用用途:

    1. 通过在外部调用闭包函数，从而在外部访问到函数内部的变量，可以使用这种方法来创建私有变量。

    2. 保存已经运行结束的函数上下文中的变量。

     for(let i = 1,i<=5;i++){
     setTimeout(function timer(){
         console.log(i)
     },i*1000)
 }
 /*如果用var声明变量
 由于变量 i 的作用域是整个函数，setTimeout 中的闭包捕获的是相同的 i 变量，而不是循环中每次迭代时创建的新变量。

 for(var i = 1,i<=5;i++){
     ;(function(j){

             console.log(j)

    # 28.对作用域和作用域链的理解

    全局作用域

    1. 最外层函数和最外层函数外面定义的变量拥有全局作用域

    2. 所有未定义直接赋值的变量自动声明为全局作用域

    3. 所有 window 属性的对象自动拥有全局作用域

    4. 过多的全局作用域变量会污染命名空间，容易引起命名冲突。

    函数作用域

    1. 声明在函数作用域内的变量

    2. 作用域是分层的，内层作用域可以访问到外层作用域

    块级作用域

    1. let 和 const 可以创建块级作用域 ，也可以用{}创建

    2. let 和 const 声明的变量不會有变量提升，也不可以重复声明。

    3. 在循环内部比较适合绑定块级作用域，这样可以将声明的计数器变量限制在循环内部。

    作用域链

    在当前作用域中查找所需变量，但是该作用域没有这个变量，如果在自己作用域找不到该变量就去父级作用域查找，依次向上级作用域查找，直到访问到 window 对象就被终止，这一层层的关系就是作用域链。通过作用域链，可以访问到外层环境的变量和函数。

    # 29.对执行上下文的理解？

    执行上下文类型

    全局执行上下文:任何不在函数内部的都是全局执行上下文，它首先会创建一个全局的 window 对象，并设置 this 值为全局对象。

    函数执行上下文:当一个函数被调用时，就会为该函数创建一个新的执行上下文，函数的上下文可以有任意多个。

    执行上下文栈

    js 使用全局上下文栈进行来管理上下文，每当遇到一个函数调用，就创建一个新的执行上下文压入栈顶，js 会执行位于栈顶的函数，当函数执行完成之后，上下文会从栈中弹出，继续执行下一个上下文。

    console.log("1");
    second();
    console.log("2");
}

    console.log("3");
}
first(); // 1 3 2

//解析的时候会先创建一个全局执行上下文环境，先把代码中即将执行的变量、函数声明都拿出来，
//变量先赋值为undefined，函数先声明好可使用。
//这一步执行完了，才开始正式的执行程序

    # 30.this/call/apply/bind 的理解？

    对 this 对象的理解

    • 在函数中直接调用指向 window

    • 在定时器延时器中直接调用也指向 window

    • 在对象方法中调用指向调用该方法的对象

    • 在构造函数中调用指向通过该构造函数实例化出来的对象。

    call/apply/bind

    • apply 接受两个参数 ，一个是新的 this 指向，一个是参数组成的数组

    • call 接受的第一个参数为新的 this 指向，剩下的为剩余参数。

    • bind 第一个参数为新的 this 指向，后面的参数依次传入，不会立即执行。

    # 31.异步编程的实现方式

    1. 回调函数。多个回调函数嵌套时会造成回调地狱问题。不利于代码维护。

    2. Promise。可以进行链式调用，但是链式调用过多也会造成代码阅读性问题。

    3. generator。 在函数执行过程中将执行权转移出去，当函数执行完毕再将执行权转移回来，因此需要有一个自动执行 generator 的机制。

    4. async/await 。Promise 的语法糖，可以将异步代码当做同步代码来书写。

    # 32.对 Promise 的理解。

    Promise 本身是同步的立即执行函数。

    Promise 的实例有三种状态:

    1. Pending(进行中)

    2. resolved(已完成)

    3. rejected(已拒绝)

    一旦状态改变，就无法在改变。

    缺点:

    1. 无法取消 Promise，一旦新建就会立即执行，无法中途取消。

    2. 不设置回调函数无法抛出错误

    3. 当处于 pending 状态时，无法判断目前进展到哪一个阶段。

    // 创建promise的方法
const p = new Promise(function (resolve, reject) {
    if ("异步操作成功") {
        resolve();
    } else {
        reject();
    }
});

    # 33.async/await 的理解

    async/await 是 generator 的语法糖，async 用来声明异步函数，await 用来等待结果。async 函数返回一个 promise，所以在最外层不能 y 用 await 获取返回值时，应该用.then 来处理。await等待的是一个 promise 的结果

    async/await 对比 Promise 的优势。

    1. 代码阅读起来更加同步。

    2. Promise 传递中间值非常麻烦，async/await 几乎是同步的写法。

    3. 错误处理友好，可以使用 try/catch 语法

    4. 调试友好

    # 34.对象创建的方式有哪些？

    1. 工厂模式，用函数封装创建对象的细节，通过调用函数来达到复用的目的。

    2. 构造函数模式，执行构造函数会先创建一个新的对象，然后将对象的原型指向构造函数的 prototype，然后将执行上下文中的 this 指向这个对象，最后在执行整个函数，如果返回值不是对象，则返回新建的对象。

    3. 原型模式，每一个函数都有一个 prototype 属性，这个属性是一个对象，通过构造函数创建的所有实例都能够共享里面的属性和方法，因此可以使用原型对象来添加公用属性和方法来实现代码的复用。

    4. 构造函数加原型模式。通过构造函数来初始化对象属性，通过原型对象来实现函数方法的复用。

    2. 借用构造函数继承：缺点无法实现函数方法的复用。

    # 36.浏览器的垃圾回收机制

    概念:js 代码运行时，需要分配内存空间来存储变量和值。当变量不在参与运行时，就需要系统收回被占用的内存空间，这就是垃圾回收。

    回收机制:

    1. 全局变量的生命周期会持续到页面卸载。

    2. 局部变量在函数执行结束后就会被释放，闭包情况下，变量不会被自动回收。

    回收的方式:

    1. 标记清除法：当变量进入执行环境时，就标记这个变量"进入环境"，被标记为进入环境的变量是不能被回收的，因为他们正在被使用，当变量离开环境时，会被标记为离开环境，被标记为离开环境的变量会被清除，回收内存。

    2. 引用计数法:当声明了一个变量并将引用类型赋值给该变量时，这个值的引用计数就是 1，如果包含这个值引用的变量又取得了另外一个值，引用次数就减一，当引用计数变为 0 时，这个变量占用的内存空间就会被释放出来。

    function fun() {
    let obj1 = {};
    let obj2 = {};
    obj1.a = obj2; // obj1引用obj2
    obj2.a = obj1; // obj2引用obj1
}
// 这种情况下 ，就需要手动释放内存
obj1.a = null;
obj2.a = null;

    减少垃圾回收:

    1. 当代码比较复杂时，垃圾回收带来的代价比较大，所以应该尽量 j 减少垃圾回收。

    2. 对数组进行优化: 清空数组时，赋值为【】会创建一个新的空对象数组，可以设置 length 为 0 来清空数组。

    3. 对对象进行优化：尽量复用对象，如果对象不在使用，就将其设置为 null

    4. 对函数进行优化:函数中的循环表达式，如果可以服用，尽量放在函数的外面。

    # 37.哪些情况会导致内存的泄露？

    1. 意外的全局变量:使用了未声明的全局变量，从而创建了一个全局变量，该变量一直存在在内存中，无法被回收。

    2. 未清除的计时器和延时器

    3. 脱离 dom 的引用:获取了一个 dom 的引用，而后面这个元素被删除，由于一直保留了对这个元素的引用，所以它也无法被回收。

    4. 闭包：不合理的使用闭包，从而导致某些变量一直被留在内存当中。
