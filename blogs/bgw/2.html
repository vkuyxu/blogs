    # 1.css 选择器及其优先级。

    1. id 选择器 ：#id 权重 100

    2. 类选择器:.classname 权重 10

    3. 属性选择器：a[ref='eee'] 权重 10

    4. 伪类选择器:li:last-child 权重 10

    5. 标签选择器:div 权重 1

    6. 伪元素选择器: li：after 权重 1

    7. 相邻兄弟选择器： h1 + p 权重 0

    8. 子选择器 ： ul>li 权重 0

    9. 后代选择器 ： li a 权重 0

    10. 通配符选择器 ： * 权重 0

    #内联样式>内部样式>外部样式>浏览器用户自定义样式>浏览器默认样式

    # 2.css 中可继承和不可继承的属性有哪些。

    # 一.无继承性的属性

    1. dispaly：规定元素生成的框类型

    2. 文本属性：

    vertical-align:垂直文本对齐

    text-decoration：规定添加到文本的装饰

    text-shadow：文本阴影效果

    white-space：空白符的处理

    unicode-bidi：设置文本的方向

    1. 盒模型属性:width ,height ,margin , border ,padding

    2. 背景属性:background,background-color,background-image

    3. 定位属性:float ,position ,left ,right ,z-index,clear

    4. 生成内容属性:content，counter-reset

    5. 轮廓样式属性:outline-style , outline-width , outline-color, outline

    # 二.有继承性的属性

    1. 字体系列的属性 font-family : 字体系列 font-weight:字体的粗细 font-size:字体的大小 font-style:字体的风格 2. 文本系列的属性 text-indent:文本缩进 text-align:文本水平对齐 line-height:行高 word-spacing: 单词之间的间距 letter-spacing:中文或者字母之间的间距 text-transform:控制文本大小写

    2. color:文本颜色

    3. 元素可见性:visibility 控制元素显示隐藏

    4. 列表布局属性： list-style：列表风格

    5. 光标属性:cursor:光标显示为何种形态

    # 3.dispaly 的属性及其作用

    1. dispaly:none 元素不显示，并且会从文档流中移除

    2. dispaly:block 块类型 默认为父级宽度，可设置宽高，换行显示

    3. display:inline 行内元素 默认为内容宽度 ，不可设置宽高，同行显示

    4. dispaly:inline-block 默认宽度为内容宽度，可以设置宽高，同行显示

    5. dispaly：list-item 像块类型元素一样显示，并添加样式列表标记

    6. display:table 会作为块级表格来显示

    7. display:inherit 从父元素继承 display 属性的值

    # 4.display 的 block，inline ，和 inline-block 的作用

    1. block ：会独占一行，多个元素会另起一行，可以设置宽高，padding，margin 四个方向都占位

    2. inline：元素不会独占一行，同行显示，不可以设置宽高，padding，margin 在水平方向占位，垂直方向不占位

    3. inline-block ：元素不会独占一行，同行显示，可以设置宽高，padding，margin 四个方向都占位

    行内元素：

    1. 设置宽高无效，padding，margin 在水平方向占位，垂直方向不占位

    2. 不会自动换行

    块级元素：

    1. 可以设置宽高，padding，margin 在四个方向都占位。独占一行

    # 5.隐藏元素的方法有哪些？

    1. display：none 元素不会渲染，在页面中不占位，也不會有点击事件。

    2. visibility：hidden 元素在页面中依然占位，但是不会绑定响应的点击事件。

    3. opacity ：0 将元素透明度设置为 0，用来实现元素的隐藏。占位，且能响应点击事件。

    4. position：absolute 绝对定位， 将元素移除可视区域内，以此来实现元素的隐藏

    5. z-index 负值 ， 使其他元素来覆盖住该元素，实现元素的隐藏

    6. clip/clip-path 使用元素裁剪的的方式来实现元素的隐藏，占位，不会绑定事件。

    7. transform ：scale(0.0) 将元素缩放设置为 0，来实现元素的隐藏，占位 ，不会绑定事件。

    # 6.link 和@import 的区别？

    都是引用外部 css 的一种方式

    1. link 除了加载 css 以外，还可以定义 Rss 等其他事物，@import 属于 css 范畴，只能加载 css

    2. link 引用 css 时，与页面并行加载，@import 需要等网页加载完毕后加载

    3. link 是 XHTML 标签 ，无兼容性为题，@import 是 css2.1 提出的 低版本的浏览器不兼容

    4. link 支持用 js 去控制 dom 的样式，而@import 不支持。

    # 7.transition 和 animation 的区别？

    transition 是过渡属性，强调过渡。需要触发一个事件才去执行动画。需要设置一个开始关键帧和结束关键帧

    animation 是动画属性，不需要触发事件，设置好时间之后可以自己执行，且可以循环一个动画。但是可以设置多个关键帧

    # 8.display：none 和 visibility：hidden 的区别

    display：none 会让元素从页面消失，不占位，是非继承属性，修改 display 属性会造成文档的重排。

    visibility：hidden ，占位，是继承属性，可以通过 visibility：visible 可以让子孙节点显示。修改 visibility 属性会造成重绘

    # 9.伪元素和伪类的区别和作用？

    冒号是伪类 双冒号是伪元素

    伪类是通过在元素选择器上加入伪类改变元素状态

    伪元素通过对元素的操作进行对元素的改变。

    # 10.盒模型的理解。

    由四个部分组成，margin，padding，border，content

    分为标准盒模型和怪异盒模型：设置 box-sizing：content-box/border-box

    标准盒模型的 width 和 height 属性只包含 content

    IE 盒模型的 width 和 height 属性的范围包含了 border，padding，content

    # 11.为什么有时候用 translate 来改变位置而不是定位?

    translate 是 transform 的一个属性值，改变 transform 或者 visibility 不会触发重绘，只会触发复合。

    定位会触发重新布局，进而触发重绘和复合

    重绘:元素样式属性发生更改，需要重新绘制受影响的元素。

    复合:将新的元素的绘制结果合并到页面中。

    # 12.CSS3 中有哪些新特性？

    1. 新增各种 css 选择器

    2. 圆角 border-radius

    3. 多列布局 multi-column layout

    4. 阴影和反射 shadowelfect

    5. 文字特效 text shadow

    6. 文字渲染 text-decoration

    7. 线性渐变 gradient

    8. 旋转 transform

    # 13.对 CSSSprites 的理解（精灵图）

    将一个页面所涉及到的图片全都包含到一张大图中去，然后利用 css 的 background-position 属性进行定位。

    优点：减少网页的 http 请求，提高了页面性能。

    缺点:开发时比较麻烦。维护时也比较麻烦。需要修改 css

    # 14.什么是物理像素，逻辑像素和像素密度，为什么在移动端开发时需要用到@3x,@2x 这种图片？

    物理像素是屏幕上实际的物理像素。是显示设备上最小的显示单元。

    逻辑像素是用来开发 web 页面的 css 像素

    像素密度是指单位物理长度内包含的像素数，像素密度越高，屏幕就有更高的清晰度和细节度。

    为了适配移动端，不同的设备像素密度是不同的，为了在不同的设备上获得一致的显示效果，需要用到@3x @2x 这种图片

    # 15.margin 和 padding 的使用场景

    需要在 border 外侧添加空白且不需要背景色时。使用 margin

    需要再 border 内侧添加空白且需要背景色时，使用 padding

    # 16.line-heihgt 的使用方式。

    指文本的高度，如果标签没有定义高度，则最终高度由 line-height 决定

    把 line-height 的高度设置的和 height 一样，可以实现单行文字的水平垂直居中。

    line-height 的赋值单位:

    带单位:px 是固定值，而 em 则会参考父元素 font-size 计算自身的行高。

    纯数字:会把比例传给后代。例如，父级行高为 1.5，子元素字体为 18px，则子元素行高为 1.5* 18 =27px

    百分比: 将计算后的值传递给后代

    # 17.CSS 优化和提高性能的方法有哪些?

    加载性能：

    1. css 压缩，将写好的 css 进行打包压缩，可以减少文件体积

    2. css 单一样式，尽量少写复合样式。效率更高

    3. 减少使用@import 多使用 link，因为 link 与页面并行加载

    选择器性能：

    1. 尽量少在通配符中书写属性。

    2. 尽量精准的选中元素，从而过滤到无关的规则，就是尽量用 class 选择器而不是后代选择器

    3. 了解哪些属性是可以继承的，从而少书写相同的属性

    渲染性能：

    1. 慎用高性能属性:浮动，定位

    2. 尽量减少页面的重排与重绘

    3. 属性为 0 时不加单位

    4. 属性值为 0.xxx 时可以省略 0

    5. 正确使用 display 属性

    6. 不滥用 web 字体。

    可维护性，健壮性：

    1. 将具有相同属性的 css 样式抽离出来，整合并通过 class 在页面中使用，提高 css 的可维护性

    2. 样式与内容分离

    # 18.css 预处理器/后处理器是什么?为什么要使用它们？

    预处理器:less,sass 用来预编译 less 和 sass ，增加了 css 代码的复用性。

    后处理器:如 ：postCss 通常是在完成的样式表中根据 css 规范处理 css，让其更加有效，最常用的是给

    css 添加浏览器前缀，实现跨浏览器兼容性问题。

    # 19.display：inline-block 什么时候会显示间隙？

    有空格的时候会显示间隙，可以删除空格解决

    margin 为正值时，可以修改 margin 为负值

    使用 font-size 时，可通过设置 font-size:0 letter-spacing word-spacing 来解决

    # 22.如何判断元素是否达到可视区域？

    window.innerHeight 是浏览器可视区的高度

    document.body.scrollTop || document.documentElement.scrollTop 是浏览器滚动过的距离

    imgs.offsetTop 是元素距离文档顶部的距离，包括滚动条的距离

    判断内容是否达到显示区：imgs.offsetTop < window.innerHeight + document.body.scrollTop

    # 23.响应式设计的基本概念和原理

    响应式设计是一种用于创建适应不同设备和屏幕尺寸的网页布局和设计的方法。它的基本概念是通过

    使用 CSS 媒体查询、相对单位和弹性布局等技术，使得网站能够根据设备的特性和视口的大小变化进

    行动态调整，以提供更好的用户体验。

    下面是响应式设计的一些基本原理和概念：

    1. 媒体查询（Media Queries）：媒体查询是 CSS3 中的一个功能，它可以根据设备的特性（如屏幕尺寸、屏幕方

    向、分辨率等）应用不同的 CSS 样式。通过使用媒体查询，可以针对不同的设备设置适当的布局和样式，从而

    实现响应式设计。

    1. 流式布局（Fluid Layout）：流式布局是指通过使用相对单位（如百分比、em、rem 等）来定义元素的尺寸和间

    距，使得页面的布局可以根据视口的大小进行自适应调整。流式布局可以使得网站在不同设备上保持比例和平滑

    的流动效果。

    1. 弹性图片（Flexible Images）：响应式设计还包括对图片的适应。通过使用 CSS 样式（如 max-width:

    100%），可以使得图片在各种屏幕尺寸上自动调整大小，避免图片在小屏幕上显示过大而破坏布局。

    1. 断点（Breakpoints）：为了更精确地适应不同的设备和屏幕尺寸，响应式设计通常会使用断点来调整布局和样

    式。通过设置不同的断点，可以在不同的屏幕尺寸下应用特定的 CSS 样式，以优化布局和用户体验。

    1. 渐进增强（Progressive Enhancement）：响应式设计的原则之一是渐进增强。即使用先进的技术和功能来为现

    代设备提供更好的用户体验，同时确保基本内容能够在不支持这些功能的旧设备上正常显示和访问。

    综合来说:响应式设计是一种灵活的网页设计方法，通过利用媒体查询、流式布局、弹性图片和断点

    等技术，使得网站能够适应不同的设备和屏幕尺寸，提供良好的用户体验。这种设计方法可以减少针

    对不同设备的重复劳动，并促进网站的可维护性和可访问性。

    # 24.对 BFC 的理解，如何创建 BFC？

    BFC 是一个独立的布局容器，在容器中按照一定规则进行物品摆放，不会影响到其他环境中的物品。

    创建 BFC 的条件:

    1. 根元素:body,

    2. 元素设置浮动：float 出 none 以外的值。

    3. 元素设置绝对定位:position(absolute，fixed)

    4. display 的值为:inline-block ,flex 等

    5. 设置 overflow 属性。

    BFC 的特点:

    1. 垂直方向上，自上而下排列

    2. 相邻的两个容器 margin 会重叠

    3. 计算 BFC 的高度时，需要计算浮动元素的高度

    4. BFC 区域不会与浮动的容器发生重叠

    5. BFC 是独立的容器，容器内部元素不会影响外部元素

    6. 每个元素的左 margin 值和和容器的左 border 相接触

    BFC 的作用:

    1. 解决了 margin 重叠问题

    2. 解决高度塌陷的问题

    3. 创建自适应两栏布局
