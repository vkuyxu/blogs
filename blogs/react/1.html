    vue 和 react 的区别🔥

    vkuyxu 2023/6/13 VueReact

    # 目录

    • 一、什么是 react 和 vue
    • 二、共同点
    • 三、不同点
    • 四、语法区别
    • 五、react 和 vue 的选择

    # 一、什么是 react 和 vue

    react

    React是由Facebook开发的用于构建用户界面的JavaScript库，React主张是函数式编程的理念。实现了 前端界面的高性能开发。很擅长处理组件化的页面。在React中，所有的组件的渲染都依靠jsx，它是 JavaScript的语法扩展。它在创建UI组件和调试时非常方便。

    vue

    Vue是一个渐进式JavaScript框架，Vue的核心库只关注视图层，易于上手，可以与第三方库或既有项目整合。

    # 二、共同点

    1. 数据驱动视图
    2. 组件化
    3. 虚拟 DOM
    4. 单向数据流
    5. 响应式数据

    # 三、不同点

    1. 核心思想不同：
    • vue的定位是尽可能的降低前端开发的门槛，让更多的开发者能够更快的上手开发，所以他的特点是：灵活易用的渐进式框架，进行数据拦截/代理，它对侦测数据的变化跟敏感、更精确。
    • react从一开始的定位就是提出 UI 开发的新思路，所以React推崇函数式编程(纯组件)，数据不可变以及单向数据流，当然需要双向的地方也可以手动实现，比如借助onChange和setState来实现。
    1. 响应式原理不同：
    • vue 依赖收集，自动优化，数据可变。vue 递归监听 data 的所有属性，直接修改。当数据改变时，自动找到引用组件重新渲染
    • react基于状态机，手动优化，数据不可变，需要setState驱动新的state替换老的state，但数据改变时，以组件为根目录，默认全部重新渲染，所以react中会需要shouldComponentUpdate来优化性能。
    1. 组件写法差异：
    • vue推荐的做法是template的单文件组件格式(简单易懂，从传统前端转过来易于理解)，即html、js、css写在一个文件中(也支持jsx语法，但官方不推荐)。
    • react推荐的做法是JSX + inline style，也就是把HTML和CSS全都写进javascript中，即all in js。
    1. diff 算法不同：
    • vue的 diff 算法是双端比较，vue在比较两组节点时，会从两端开始比较，如果两端都相同，则继续向中间比较，直到找到不同的节点。在对比节点时，当节点元素相同，但 className 不同，认为是不同类型的元素，删除重建。
    • react的 diff 算法是单向比较，react在比较两组节点时，会从父节点开始比较，如果父节点相同，则继续比较子节点，直到找到不同的节点。在对比节点时，当节点元素相同，但 className 不同，认为是同类型的元素，只是修改节点属性。
    1. 渲染过程：
    • vue的渲染过程是：template -> AST -> render -> Virtual DOM -> UI， vue可以更快的计算出Virtual DOM的差异，这是由于它在渲染过程中，会跟踪每一个组件的依赖关系，不需要重新渲染整个组件树。
    • react的渲染过程是：JSX -> JS -> Virtual DOM -> UI，ract在应用的状态被改变时，全部子组件都会重新渲染。通过shouldComponentUpdate这个方法可以进行控制，但vue视为默认行为。

    1. vuex 和 redux：

      从表面上来说，store注入和使用方式有些区别

    • vuex中，$store被直接注入到了组件实例中，因此可以比较灵活的使用：使用dispatch、commit提交更新，通过mapState或者直接通过this.$store.state.xxx获取状态。在组件中既可以disopatch actions，也可以commit updates。

    • 在 redux 中，我们每一个组件都需要显示的用connect方法把需要的props和dispatch进行连接，并且通过dispatch和mapStateToProps进行提交和获取状态。在 redux 中只能进行dispatch，不能直接调用reducer进行修改。

      从实现原理上来说：redux 使用的是不可变数据，而 vuex 使用的可变数据。

    • Redux 每次都是用新 state 替换旧 state，而 Vuex 是直接修改。Redux 在检测数据变化的时候，是通过 diff 的方式比较差异的，而 Vuex 其实和 Vue 的原理一样，是通过 getter/setter 来比较的，这两点的区别，也是因为 React 和 Vue 的设计理念不同。React 更偏向于构建稳定大型的应用，非常的科班化。相比之下，Vue 更偏向于简单迅速的解决问题，更灵活，不那么严格遵循条条框框。因此也会给人一种大型项目用 React，小型项目用 Vue 的感觉。

    1. 框架：
    • vue 本质是 MVVM 框架，由 MVC 发展而来
    • react 是前端组件化框架，由后端组件化发展而来

    # 四、语法区别

    • vue 模板语法：{{}}; react 模板语法：{}
    • vue 组件语法：<template>; react 组件语法：<div>
    • vue 事件语法：@click; react 事件语法：onClick
    • vue 属性语法：:; react 属性语法：=
    • vue 条件语法：v-if; react 条件语法：if
    • vue 循环语法：vue 循环 dom 是在 html 里面使用 v-for 指令来进行循环，react 是使用{数组.map((循环出来的数据,下表)=>{})}来进行循环的
    • vue 样式语法：:class; react 样式语法：className

    # 五、react 和 vue 的选择

    在以下情况选择 vue

    1. 最新文档和更简单的语法。
    2. 更小，更快，更灵活。
    3. 需要丰富的 HTML 模板，易于开发。

    在以下的情况选择 react

    1. 需要构建移动应用程序。
    2. 需要构建大型应用程序。
    3. 轻量级，易于版本迁移。
